# netpeek
netpeek是一个TCP通讯拦截程序，网络通讯程序开发通常需要查看发给对方的报文有没有真正发出去，或者接收方想看看对方倒底发了什么报文过来，发送或者接收之前记录到日志文件中是一个方法或者使用开发工具进行调试，这些都是效率较低的方法，如果有一个工具能够代替服务端进行侦听并将收到的报文随即转发给服务端，并且实时将双方发出的报文随即转发给对方，当一方连接断开的时候也随即切断另一方的连接，在转发双方的报文时顺便打印到屏幕上，这样就可以很方便地进行相关检查，并且只要关闭这个转发程序就可以实现链路断开的效果，再次开启又相当于网络恢复正常，这样便可以不拔网线或禁网卡就可以测试断线重连等情景，netpeek就是这样的工具。
## 用法
netpeek [-nqsc] lsnaddr servaddr
例：
netpeek 0.0.0.0:80 baidu.com:80
### -n 选项
不显示报文，但是显示连接情况
### -q 选项
不显示任何信息
### -s 选项
后面的两个地址均为侦听端口，这样netpeek可以以服务端的角色连接两个客户端，即C1 ==> netpeek <== C2
### -c 选项
后面的两个地址均为服务端地址，这样netpeek可以以客户端的角色连接两个服务端，即S1 <== netpeek ==> S2
# ctpping
ctpping是一个CTP协议层ping工具，利用行情API接口的登录报文检测交易前置、行情前置的连通情况，如果链路通畅，则会显示连接并登录成功，还会显示响应时延，如果被防火墙拦截，不会有任何显示，如果CTP前置没开，也不会有任何显示，可以配合telnet检测端口是否正常。CTP的交易前置及行情前置均采用同一套技术，两者的登录协议是一致的，而交易的登录协议需要进行认证等，行情的登录则比较简单，这个工具不需要验证用户名密码是否正确，目的只在检测通讯是否正常，因此使用行情的登录来检测即可达到目的。
## 用法
ctpping address
例：
ctpping tcp://180.168.146.187:10100
